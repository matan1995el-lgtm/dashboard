# לוח הבקרה לאפליקציות האישיות

## מטרות הפרויקט ותיאור כללי
- **שם הפרויקט:** לוח הבקרה לאפליקציות האישיות
- **מטרה:** לספק מסך אחד נוח שמרכז את כל האפליקציות/המערכות שבנית, כולל קפיצה מהירה לכל פרויקט עם **סנכרון אוטומטי בענן**.
- **תכונות עיקריות:**
  - הצגה ויזואלית של כרטיסי אפליקציות עם שם, תיאור, שפה, קטגוריה ותגיות.
  - חלון קופץ להוספת/עריכת אפליקציה בלחיצה אחת.
  - סינון לפי שפה וקטגוריה + חיפוש טקסט חכם.
  - **סנכרון אוטומטי בענן** - כל השינויים נשמרים ב-RESTful Table API ומסונכרנים בין כל המכשירים.

## תכונות שהושלמו כרגע
- ממשק רספונסיבי בעברית (RTL) עם עיצוב מודרני וממוקד - **מותאם לחלוטין למובייל וטאבלט**.
- **🎨 דף הגדרות מתקדם** - שליטה מלאה על כל היבט בעיצוב:
  - ערכות נושא מוכנות (ברירת מחדל, כהה, כחול, ירוק) + אפשרות ליצור מותאם אישית
  - עריכת כל הצבעים (ראשי, משני, רקע, כרטיסים, טקסט, גבולות וכו')
  - שליטה מלאה בטיפוגרפיה (גופן, גדלים, משקל, גובה שורה)
  - התאמת מרווחים ועיגולים (פינות, ריפוד, מרווחים, צללים)
  - עריכת כל הטקסטים במערכת (כותרות, כפתורים, הודעות)
  - תצוגה מקדימה חיה של השינויים
  - ייצוא/ייבוא הגדרות לשמירה ושיתוף
- **הוספת אפליקציה חדשה** - טופס מלא עם שם, תיאור, קישור, שפה, קטגוריה, תגיות וצבע כרטיס.
- **עריכת אפליקציה קיימת** ✏️ - לחיצה על כפתור העריכה בכרטיס תפתח את הטופס עם הנתונים הקיימים.
- **מחיקת אפליקציה** 🗑️ - מחיקה מהירה עם אישור משתמש.
- **סנכרון ענן אוטומטי** ☁️ - כל שינוי נשמר מיידית בענן ומסונכרן בין כל המכשירים (נייד, מחשב, טאבלט).
- **תזכורת גיבוי קומפקטית** 💾 - הודעה קטנה וחכמה בראש הדף שמופיעה כל 12 שעות ומזכירה לייצא גיבוי ל-Google Drive/OneDrive.
- **ייצוא נתונים** 📥 - הורדת כל האפליקציות לקובץ JSON (גיבוי) - ישירות מכפתור התזכורת או מהכותרת.
- **ייבוא נתונים** 📤 - טעינת אפליקציות מקובץ JSON (שחזור/העברה).
- סטטיסטיקות חיות (סה"כ אפליקציות, מספר שפות פעילות, מספר קטגוריות פעילות).
- חיפוש וסינון חכם לפי טקסט חופשי, שפה וקטגוריה.
- **סטטוס סנכרון קבוע** - תצוגה תמידית של מצב הסנכרון (מוכן, מסונכרן, טוען, שגיאה) שלא נעלמת.
- עיצוב מותאם נייד: כרטיסים ברוחב מלא, כפתורים נוחים, טפסים מותאמים למסכים קטנים.
- תמיכה ב-iPhone Chrome - כל העיצוב מוטמע בתוך ה-HTML לטעינה מהירה ואמינה.

## מסלולי כניסה קיימים (URIs)
- `/index.html` – עמוד אחד המשמש כלוח הבקרה. אין פרמטרים מיוחדים.
- `/settings.html` – דף הגדרות מתקדמות לעיצוב והתאמה אישית מלאה.

## תכונות שעדיין לא יושמו
- אפשרות גרירה ושחרור (Drag & Drop) לשינוי סדר הכרטיסים.
- מצב "מועדפים" או סימון כוכבים להדגשת אפליקציות חשובות.
- מיון מתקדם (לפי שם, תאריך, קטגוריה).
- תמונות/אייקונים לכל אפליקציה במקום רק צבע.

## צעדים מומלצים להמשך פיתוח
1. להוסיף מצב "מועדפים" (⭐) או תגית מיוחדת להדגשת אפליקציות חשובות.
2. להוסיף אפשרות גרירה ושחרור לסידור ידני של הכרטיסים.
3. להוסיף מיון מתקדם (לפי שם א-ב, תאריך יצירה, קטגוריה).
4. להוסיף תמונות/אייקונים לכל אפליקציה במקום רק צבע.
5. להוסיף התראות push כאשר אפליקציה חדשה נוספה ממכשיר אחר.

## כתובות ציבוריות ו-API
- **Production URL:** טרם פורסם. כדי לפרסם ולשתף, עבור ללשונית **Publish** בפרויקט.
- **API endpoints:** המערכת משתמשת ב-**RESTful Table API** לסנכרון בענן:
  - `GET/POST tables/apps` - קריאה ויצירת אפליקציות
  - `PUT/DELETE tables/apps/{id}` - עדכון ומחיקת אפליקציות

## מודל הנתונים ואחסון
הנתונים נשמרים ב**RESTful Table API בענן** ומסונכרנים אוטומטית בין כל המכשירים. כל אפליקציה מיוצגת באובייקט עם המאפיינים הבאים:

| שדה        | סוג            | תיאור                                                 |
|------------|----------------|---------------------------------------------------------|
| `id`       | string         | מזהה ייחודי (UUID, עם fallback במידה והדפדפן לא תומך) |
| `name`     | string         | שם האפליקציה כפי שיוצג בכרטיס                         |
| `description` | string      | תיאור חופשי                                           |
| `url`      | string (URL)   | קישור ישיר לאפליקציה                                  |
| `language` | string         | שפת הפיתוח המרכזית                                    |
| `category` | string         | קטגוריה כללית/נושא                                    |
| `tags`     | array<string>  | תגיות מופרדות בפסיק                                   |
| `color`    | string (HEX)   | צבע הכרטיס                                             |
| `createdAt`| number (ms)    | תאריך יצירה למיון יורד                                |
| `updatedAt`| number (ms)    | תאריך עדכון אחרון (מתעדכן בעריכה)                   |

## איך הסנכרון עובד?
1. **טעינה ראשונה:** הדף טוען את כל האפליקציות מה-API בענן
2. **הוספה:** אפליקציה חדשה נשמרת מיידית ב-API ומתעדכנת בכל המכשירים
3. **עריכה:** שינויים נשמרים באופן מיידי ומסונכרנים
4. **מחיקה:** מחיקה מתבצעת בענן ומתעדכנת בכל מקום
5. **סטטוס חי:** המערכת מציגה הודעה כאשר פעולה מצליחה או נכשלת
